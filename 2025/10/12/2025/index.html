2025 月度记录 其九

发表于2025-10-12|更新于2025-10-12
|浏览量:

September

Sep.25th

  • 补充了 C++ 中有关字符串的一些小知识：
    • 传参时的空格问题 - std::ws 以及 cin.ignore()
    • 作为入口参数时使用 std::string_view 来节省性能开销
    • 了解了一下字面值 std::string_literal，一般可以直接将 C 风格字符串直接转为 string 对象，但我觉得可能不是很常用。

Sep.26th

  • 补充了 C++ 中有关变量和运算的一些小知识：

    • 静态转换 static_cast （对比强制转换）
    • 常量表达式 constexpr 与 const 的适用场合，我暂时没想到我该怎么用。
    • 位运算对象 bitset （不错）

  • 了解了命名空间 namespace，包括其与子级的创建与调用。

  • 前向声明 ( forward declartion ) 允许不定义函数就调用，但是包含这个函数定义的文件必须也参与编译。

  • inline 修饰符允许函数在单独的头文件中直接 定义， 并且被多个 cpp 文件调用。 在旧版本 C++ 中，这个修饰符的作用是将函数调用改为直接执行函数的代码段以节省开销，现代 C++ 中已经可以自动优化了。

    同时，内联的子命名空间 inline namespace 会自动归入父级，所以可以写一个内联的空间作为当前使用的版本，然后再写一个不内联的同名空间 namespace 作为新版本，这样可以进行版本控制。

  • using ：using 声明只引入命名空间中的单个名字，而 using 指令将整个命名空间引入，污染作用域。

Sep.27th

  • 学习了 C/C++ 中的随机数生成算法，C++ 的 random 库更加安全灵活。

  • 粗略地了解了一下 assert 以及 static_assert ，我感觉我用不到。使用场景上有点 const 以及 constexpr 的感觉。

  • 编译器的隐式转换（promotion / conservation）：用 {} 初始化避免窄化、或是用显式转换来强制进行。运算时两数类型不同时，用 std::common_type_t< > 来获取最终隐式转换后的共同类型。

  • typeid(expr) 可以返回表达式的类型，如果只是想知道的话。

  • 补充了 using 之于 typedef 的替代效果。

  • auto 类型 - 自动推导类型。

    • 变量以 字面量 / 算式 / 变量声明的时候，auto 自动推导出其应当拥有的类型。
    • 函数用 auto 自动推导返回值的类型，比如 auto add(int x,int y){return (x+y);} ，这时候 函数的返回值类型必须唯一 。
    • 推导常量的类型时，会丢弃 const / constexpr , 需要显式地在 auto 前自己修饰。
    • 推导字符串 / string_literal / string_view
    • 当函数使用尾置返回类型时，auto 是语法要求的一部分。尾置返回类型的目的是将复杂返回类型用 -> 写在函数名后，方便阅读。

  • 函数重载 - 类似语法糖的玩意，允许重名但是参数有区别的函数，比如参数的类型、数目以及顺序。只有返回值类型的不同 不会被编译器认为是函数重载。
