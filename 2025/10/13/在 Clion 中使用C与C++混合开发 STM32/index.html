1. 混合C / C++

1.1. C 与 C++ 的相互包含关系

C++ 可以包含 C 的头文件与代码，因为其本身类似 C 的超集，然而对于一个 C 文件，其无法直接引用 C++ 的头文件或者函数，当编译一个 C 文件的时候，由于 gcc 不能识别 C++ 语法，所以会报错。

比如，main.c 引用了含有 C++ 语法的 mycpp.h 时就会出现错误。

  • 任何包含 C++ 特性的 函数 / 语句 / … 等内容不能够在 C 文件中 直接 使用
  • C++ 内容如果需要在 gcc 下通过编译1，必须使用 #ifdef __cplusplus 条件来保护

但是既然这是在一个 C++ 头文件内，那么我们肯定希望这个 C++ 文件也能被 g++ 编译，否则 C++ 特性就没有用处了。

假如 mycpp.h 同时被 mycpp.cpp 包含，那么在其编译单元内 mycpp.h 就会被 g++ 编译，此时纯 C 的函数 dosomething() 就会出现一些情况：

  • 对于 foo() ，这个函数不可能被 gcc 编译，所以 g++ 在处理它的时候会正确地将其修饰为 C++ 名称 _Z3foov (name mangling) 以支持函数重载特性。

  • 对于 dosomething() , 这个函数既要被 gcc 编译，也要被 g++ 编译，这就导致了最后其修饰出来的名称不匹配（因为 g++ ），链接器就会报错。

所以我们的核心诉求就是让 g++ 在编译 dosomething() 的时候，生成与 gcc 一致的名称。

2.4 添加其他文件

如果需要添加其他 C/C++ 外设的文件：

  • 如果是混合使用的 C 文件，需要用 ifdef __cplusplus 与 extern"C"{} 来保护函数，然而
  • 如果是 C++ 文件，那么由于其只在 cppmain.cpp 下使用，所以只需要注意其头文件在插入 cppmain.h 的时候用 ifdef __cplusplus 保护。
  • 无论何种类型，都需要在 CMakeLists.txt 中配置好路径。
